## Context and Objective

This repository implements the **cryptographic layer** of the verifiable anonymous electronic voting system. The central artifact is a single Circom circuit — `voter_proof.circom` — that anonymously proves:

1. The voter holds a `voter_id` whose Poseidon hash is present in the authorized voter Merkle tree.
2. The nullifier was computed correctly and deterministically.
3. The vote is cast for a specific candidate, race, and election.

The compiled artifacts (`.wasm`, `.zkey`, `Verifier.sol`, `verification_key.json`) feed the other three repositories in the project.

---

## Fixed Architectural Decisions 🔒

These decisions **cannot be changed** without a full team review. If any task appears to require a change here, stop and consult.

| Decision | Value |
|---|---|
| Proving protocol | **PLONK** (never Groth16) |
| Hash function | **Poseidon** (circomlib — never SHA-256, Pedersen, or any other) |
| Merkle tree depth | **4** (supports up to 16 leaves → 15-voter PoC) |
| Nullifier formula | `Poseidon(voter_id, election_id, race_id)` — 3 inputs, in this order |
| Number of public signals | **5** (merkle_root, nullifier_hash, candidate_id, election_id, race_id) |
| Cryptographic curve | **BN128 (bn254)** |
| Powers of Tau | Pre-existing Hermez Network ptau (universal; not generated by the team) |
| Compiler | `circom` 2.x, flags `--r1cs --wasm --sym --c` |

## Phase 1 — Circuit Implementation

### 1.1 File: `circuits/voter_proof.circom`

Implement the complete circuit as specified below. Read each section before writing.

#### 1.1.1 Header and imports

```circom
pragma circom 2.1.6;

include "circomlib/circuits/poseidon.circom";
include "circomlib/circuits/merkleProof.circom";
```

> **Note:** The relevant circomlib template for Merkle inclusion proof is `MerkleProof` (or `MerkleTreeChecker` depending on the installed version). Check the exact name in `node_modules/circomlib/circuits/` before including. Use whichever is available.

#### 1.1.2 Signal declarations

```circom
template VoterProof(depth) {

    // ─── PRIVATE INPUTS ──────────────────────────────────────────────
    // Never leave the device. Never become public signals.
    signal input voter_id;                      // CPF or voter registration (digits only)
    signal input merkle_path[depth];            // Sibling hash path
    signal input merkle_path_indices[depth];    // 0 = left, 1 = right

    // ─── PUBLIC INPUTS ───────────────────────────────────────────────
    // Visible on-chain. Order matters — reflected in pubSignals[0..4].
    signal input merkle_root;       // pubSignals[0] — Merkle tree root
    signal input nullifier_hash;    // pubSignals[1] — Poseidon(voter_id, election_id, race_id)
    signal input candidate_id;      // pubSignals[2] — chosen candidate (0=blank, 999=spoiled)
    signal input election_id;       // pubSignals[3] — election identifier
    signal input race_id;           // pubSignals[4] — race/office identifier
```

> 🔒 **Invariant:** `race_id` MUST be the fifth public signal (`pubSignals[4]`). Without it as a public signal, a malicious relayer could reuse a proof from one race in another. This was the relay attack vulnerability fixed in spec v4.0.

#### 1.1.3 Leaf hash computation

```circom
    // ─── VOTER_ID HASH (Merkle tree leaf) ────────────────────────────
    // The leaf is not voter_id itself — it is the Poseidon hash of voter_id.
    // This matches what is stored in the contract (voterHashes[]).
    component leafHasher = Poseidon(1);
    leafHasher.inputs[0] <== voter_id;
    // leafHasher.out is the voter_id hash — used as the leaf in the Merkle tree
```

#### 1.1.4 Merkle tree membership verification

```circom
    // ─── MERKLE TREE VERIFICATION ────────────────────────────────────
    // Proves that leafHasher.out is in the tree with root merkle_root.
    component merkleChecker = MerkleProof(depth);  // or MerkleTreeChecker(depth)
    merkleChecker.leaf <== leafHasher.out;
    merkleChecker.root <== merkle_root;
    for (var i = 0; i < depth; i++) {
        merkleChecker.pathElements[i] <== merkle_path[i];
        merkleChecker.pathIndices[i]  <== merkle_path_indices[i];
    }
    // The MerkleProof/MerkleTreeChecker template adds the internal constraints.
    // No additional === is needed for root verification.
```

> **Agent note:** Verify the exact signal names of the template (`pathElements` vs `path`, `pathIndices` vs `pathIndex`) by reading the source in `node_modules/circomlib/circuits/`. Adapt accordingly.

#### 1.1.5 Nullifier computation and binding constraint

```circom
    // ─── NULLIFIER ───────────────────────────────────────────────────
    // FIXED formula: Poseidon(voter_id, election_id, race_id)
    // 🔒 Do not change the order or number of inputs without architectural review.
    component nullifierHasher = Poseidon(3);
    nullifierHasher.inputs[0] <== voter_id;
    nullifierHasher.inputs[1] <== election_id;
    nullifierHasher.inputs[2] <== race_id;

    // The nullifier computed in-circuit MUST equal the public nullifier_hash.
    // This constraint prevents a prover from forging a different nullifier.
    nullifier_hash === nullifierHasher.out;
```

> 🔒 **Critical invariant:** `nullifier_hash === nullifierHasher.out` is a real R1CS constraint (operator `===`), NOT an `assert()`. This mathematically binds the public signal to the internal computation. Without this line, a prover could supply any nullifier_hash without computing it correctly.

#### 1.1.6 Template closure and main component declaration

```circom
    // ─── CANDIDATE_ID ─────────────────────────────────────────────────
    // candidate_id is a public signal but does not participate in any
    // in-circuit computation — it is passed directly to the contract.
    // To prevent the optimizer from removing this signal (unconstrained
    // public input vulnerability), add an identity constraint.
    signal candidate_id_check;
    candidate_id_check <== candidate_id * 1;

} // end of VoterProof template

// ─── MAIN COMPONENT ──────────────────────────────────────────────────────────
// All public signals declared here.
// The order determines pubSignals[0..4] in the verifier.
component main {public [
    merkle_root,        // pubSignals[0]
    nullifier_hash,     // pubSignals[1]
    candidate_id,       // pubSignals[2]
    election_id,        // pubSignals[3]
    race_id             // pubSignals[4]
]} = VoterProof(4);
```

> The `4` in `VoterProof(4)` is the Merkle tree depth from spec v4.0. If it ever needs to change (unlikely for this PoC), update here AND in the contract AND in the voter registration scripts.

### 3.2 Helper: `test/helpers/merkle.js`

```javascript
const { poseidonHash } = require("./poseidon");

/**
 * Builds a Merkle tree of depth `depth` from `leaves`.
 * Empty leaves are padded with the Poseidon hash of 0.
 * Returns { root, layers } where layers[0] is the leaf level.
 */
async function buildMerkleTree(leaves, depth) {
  const size = 2 ** depth;
  const ZERO_LEAF = await poseidonHash([BigInt("0")]);

  const paddedLeaves = [...leaves];
  while (paddedLeaves.length < size) {
    paddedLeaves.push(ZERO_LEAF);
  }

  const layers = [paddedLeaves];
  let currentLayer = paddedLeaves;

  for (let i = 0; i < depth; i++) {
    const nextLayer = [];
    for (let j = 0; j < currentLayer.length; j += 2) {
      const hash = await poseidonHash([
        BigInt(currentLayer[j]),
        BigInt(currentLayer[j + 1]),
      ]);
      nextLayer.push(hash);
    }
    layers.push(nextLayer);
    currentLayer = nextLayer;
  }

  return { root: currentLayer[0], layers };
}

/**
 * Generates the Merkle proof path (siblings + indices) for the leaf at `leafIndex`.
 */
async function getMerklePath(layers, leafIndex) {
  const path = [];
  const pathIndices = [];

  let currentIndex = leafIndex;
  for (let i = 0; i < layers.length - 1; i++) {
    const isRight = currentIndex % 2 === 1;
    const siblingIndex = isRight ? currentIndex - 1 : currentIndex + 1;
    path.push(layers[i][siblingIndex]);
    pathIndices.push(isRight ? 1 : 0);
    currentIndex = Math.floor(currentIndex / 2);
  }

  return { path, pathIndices };
}

module.exports = { buildMerkleTree, getMerklePath };
```

### 3.3 Main tests: `test/voter_proof.test.js`

```javascript
const path = require("path");
const { wasm: wasmTester } = require("circom_tester");
const { poseidonHash } = require("./helpers/poseidon");
const { buildMerkleTree, getMerklePath } = require("./helpers/merkle");

const CIRCUIT_PATH = path.join(__dirname, "../circuits/voter_proof.circom");
const DEPTH = 4;

describe("VoterProof Circuit", () => {
  let circuit;

  // Shared fixtures
  const voter_id    = BigInt("12345678901");   // fictitious normalized CPF
  const election_id = BigInt("1");
  const race_id     = BigInt("1");
  const candidate_id = BigInt("13");

  before(async () => {
    circuit = await wasmTester(CIRCUIT_PATH, {
      include: path.join(__dirname, "../node_modules"),
    });
  });

  // ─── Helpers ────────────────────────────────────────────────────────────────

  async function buildValidWitness(overrides = {}) {
    const v_id   = overrides.voter_id     ?? voter_id;
    const e_id   = overrides.election_id  ?? election_id;
    const r_id   = overrides.race_id      ?? race_id;
    const c_id   = overrides.candidate_id ?? candidate_id;

    const leaf = await poseidonHash([v_id]);

    // Build tree with 5 fictitious voters; our voter is at index 2
    const other = await poseidonHash([BigInt("99999999999")]);
    const leaves = Array(16).fill(other);
    leaves[2] = leaf;

    const { root, layers } = await buildMerkleTree(leaves, DEPTH);
    const { path: mPath, pathIndices } = await getMerklePath(layers, 2);
    const nullifier = await poseidonHash([v_id, e_id, r_id]);

    return {
      voter_id:             v_id.toString(),
      merkle_path:          mPath,
      merkle_path_indices:  pathIndices,
      merkle_root:          root,
      nullifier_hash:       nullifier,
      candidate_id:         c_id.toString(),
      election_id:          e_id.toString(),
      race_id:              r_id.toString(),
    };
  }

  // ─── Tests ──────────────────────────────────────────────────────────────────

  it("happy path: valid voter generates correct witness", async () => {
    const input = await buildValidWitness();
    const witness = await circuit.calculateWitness(input, true);
    await circuit.checkConstraints(witness);
  });

  it("nullifier binding: same inputs always produce same nullifier", async () => {
    const input = await buildValidWitness();
    const w1 = await circuit.calculateWitness(input, true);
    const w2 = await circuit.calculateWitness(input, true);
    expect(w1[1].toString()).to.equal(w2[1].toString()); // pubSignals[1]
  });

  it("nullifier distinctness: different race_id produces different nullifier", async () => {
    const i1 = await buildValidWitness({ race_id: BigInt("1") });
    const i2 = await buildValidWitness({ race_id: BigInt("2") });
    const w1 = await circuit.calculateWitness(i1, true);
    const w2 = await circuit.calculateWitness(i2, true);
    expect(w1[1].toString()).to.not.equal(w2[1].toString());
  });

  it("relay attack guard: mismatched nullifier_hash and race_id fails constraint", async () => {
    const input = await buildValidWitness({ race_id: BigInt("1") });
    // Force a nullifier_hash inconsistent with race_id=1 (computed with race_id=2)
    const wrongNullifier = await poseidonHash([voter_id, election_id, BigInt("2")]);
    input.nullifier_hash = wrongNullifier;
    await expect(circuit.calculateWitness(input, true)).to.be.rejected;
  });

  it("invalid Merkle root: proof fails", async () => {
    const input = await buildValidWitness();
    input.merkle_root = "9999999999999999999999999999999999999999999999999";
    await expect(circuit.calculateWitness(input, true)).to.be.rejected;
  });

  it("tampered Merkle path: proof fails", async () => {
    const input = await buildValidWitness();
    input.merkle_path[0] = "9999999999999999999999999999999999999999999999999";
    await expect(circuit.calculateWitness(input, true)).to.be.rejected;
  });

  it("blank vote: candidate_id = 0 generates valid witness", async () => {
    const input = await buildValidWitness({ candidate_id: BigInt("0") });
    const witness = await circuit.calculateWitness(input, true);
    await circuit.checkConstraints(witness);
  });

  it("spoiled/null vote: candidate_id = 999 generates valid witness", async () => {
    const input = await buildValidWitness({ candidate_id: BigInt("999") });
    const witness = await circuit.calculateWitness(input, true);
    await circuit.checkConstraints(witness);
  });

  it("cross-verification: in-circuit nullifier matches JS-computed nullifier", async () => {
    const input = await buildValidWitness();
    const witness = await circuit.calculateWitness(input, true);
    const expected = await poseidonHash([voter_id, election_id, race_id]);
    // Index 2 corresponds to pubSignals[1] (nullifier_hash) in the witness array.
    // Verify by running: snarkjs wtns export json build/witness.wtns witness.json
    // and checking which index holds the nullifier_hash value.
    expect(witness[2].toString()).to.equal(expected);
  });
});
```

## Known Issues and How to Handle Them

| Problem | Likely Cause | Action |
|---|---|---|
| `Error: Not enough powers of tau` during PLONK setup | ptau file too small for the constraint count | Run `snarkjs r1cs info` and choose a ptau with a higher power |
| `Assert failed` during witness generation | Internal constraint not satisfied; usually wrong Merkle path or nullifier | Recompute path and nullifier using the JS helpers |
| circomspect: `Signal ... may be unconstrained` | Public signal without R1CS constraint | Add an identity constraint or restructure the circuit |
| `TypeError: Cannot read properties of undefined (reading 'inputs')` | Wrong Poseidon/MerkleChecker signal name | Inspect source in `node_modules/circomlib/circuits/` |
| Test fails with `witness does not match expected` | pubSignals order in the `main` component changed | Check `public.json` and update test array indices |
